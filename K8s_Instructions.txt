# 📌 Proyecto Final: Sistema Full‑Stack por Etapas (OBLIGATORIO)

## 🎯 Objetivo general
Construir un **sistema completo tipo “Gestor de Tareas” (Task Manager)** que incluya:
- Base de datos PostgreSQL
- Backend Node.js (Express)
- Frontend React
- API REST con CRUD completo
- Ejecución local
- Dockerización
- Despliegue final en Kubernetes (kind)

Todo debe funcionar **100% en local**, sin usar ningún Cloud Provider.

---

# 🧱 ETAPA 1 — Base de Datos (LOCAL)

### Objetivo
Diseñar y crear la base de datos **antes de Kubernetes**, entendiendo el modelo de datos.

### Requisitos
- PostgreSQL ejecutándose localmente (Docker permitido)

### Entregables
Incluye:
- Diagrama lógico simple (texto)
- SQL completo en bloques de código:
  - CREATE DATABASE
  - CREATE TABLE
  - Índices
  - INSERT de datos de ejemplo
  - Queries reales usadas por el backend:
    - SELECT
    - INSERT
    - UPDATE
    - DELETE

### Validaciones
- Conectarse a la DB
- Ejecutar queries manualmente
- Ver datos persistidos

---

# 🧱 ETAPA 2 — Backend (LOCAL, SIN DOCKER)

### Objetivo
Crear el backend **conectado a la DB local**, sin contenedores aún.

### Stack
- Node.js
- Express
- pg (PostgreSQL client)

### Funcionalidad mínima
API REST con:
- GET /health
- GET /api/tasks
- POST /api/tasks
- PUT /api/tasks/:id
- DELETE /api/tasks/:id

### Entregables
Incluye:
- Estructura de carpetas
- Código backend completo y funcional
- Archivo `.env.example`
- Conexión real a PostgreSQL
- Manejo básico de errores

### Validaciones
- Probar endpoints con curl o HTTP client
- Ver logs
- Confirmar escritura en DB

---

# 🧱 ETAPA 3 — Frontend (LOCAL)

### Objetivo
Construir el frontend y consumir el backend local.

### Stack
- React
- Fetch o Axios

### Funcionalidad mínima
- Listar tareas
- Crear tarea
- Editar tarea
- Eliminar tarea
- Manejo básico de estados y errores

### Entregables
Incluye:
- Estructura del proyecto
- Código React mínimo pero funcional
- Variables de entorno para API URL
- Build y ejecución local

### Validaciones
- Flujo completo frontend → backend → DB
- UI funcional desde navegador

---

# 🧱 ETAPA 4 — Dockerización

### Objetivo
Convertir el sistema local en **contenedores Docker**.

### Entregables
Incluye:
- Dockerfile para backend
- Dockerfile para frontend
- docker-compose.yml para desarrollo local que levante:
  - PostgreSQL
  - Backend
  - Frontend

### Validaciones
- docker compose up
- Acceso al frontend
- Persistencia de DB
- Logs de servicios
